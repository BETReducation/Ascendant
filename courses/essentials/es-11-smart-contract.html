Deploy your first smart contract

Install Aiken, copy the starter template, build it, and submit it to Cardano's testnet. Your first on-chain validator in 15 minutes.

⏱ 15 min 💻 Terminal / command line required 🧪 Testnet only — free, no real ADA needed
Home › Courses › Essentials › Smart contract
🔧 What you'll have: A deployed validator on Cardano testnet
💻 Tools needed: Terminal, Aiken CLI, a text editor
🆓 Cost: Zero — testnet ADA is free
1

Install Aiken

Aiken is Cardano's purpose-built smart contract language — simpler than Haskell/Plutus and much faster to learn. Open your terminal and run:

curl -sSfL https://install.aiken-lang.org | bash

Then restart your terminal and verify it's installed:

aiken --version
📸 Screenshot: Terminal showing aiken --version output Show the version number printed after successful install
2

Create a new Aiken project

Create a new project with:

aiken new my-first-contract cd my-first-contract

This creates a project folder with the correct structure. Open it in your text editor (VS Code works great — install the Aiken extension for syntax highlighting).

📸 Screenshot: VS Code — Aiken project structure Show the file tree with validators/, lib/, and aiken.toml visible
3

Look at the starter validator

Open validators/placeholder.ak. You'll see a simple validator that always succeeds:

validator { fn spend(_datum, _redeemer, _context) { True } }

This is the "Hello World" of Cardano smart contracts. It validates any transaction that tries to spend from it. In a real contract you'd check conditions — for now, just understanding the structure is enough.

4

Build your contract

In your terminal (inside the project folder), run:

aiken build

If successful, you'll see a plutus.json file appear in your project folder. This is your compiled contract — ready to submit to the blockchain.

📸 Screenshot: Terminal — successful aiken build output Show the build success message and the generated plutus.json file
5

Get testnet ADA and check your script address

To interact with your contract on testnet, you need testnet ADA (tADA) — it's free. Go to the Cardano Testnet Faucet at docs.cardano.org/cardano-testnet/tools/faucet, enter your wallet's testnet address, and request some tADA.

Your contract's script address (where you'd lock funds to be controlled by your validator) is derived from the plutus.json file. You can compute it with the cardano-cli — covered in the full Architect course.

✅
